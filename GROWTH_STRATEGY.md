## 3. Оптимизация воронки конверсии

### 3.1 Текущая воронка и проблемы

```
Посетитель
  |
  +-- Hero CTA "Заказать аудит" --> Telegram (start=zakazat_audit)
  +-- Navbar "SOS Аудит" --> Telegram (start=sos_audit)
  +-- "Разведка" CTA --> Telegram (start=razvedka)
  +-- "Проект" CTA --> Telegram (start=proekt)
  +-- "Броня" CTA --> Лид-форма --> Telegram Bot API --> Группа
  +-- AI-виджет --> Разговор --> (нет явного CTA)
  +-- PDF скачивание --> (нет follow-up)
  +-- "История ареста" --> iframe РИА --> (dead end)
```

**Проблемы:**
- 4 из 6 CTA ведут в Telegram -- высокий барьер для тех, кто не использует TG активно
- PDF скачивание -- dead end, нет email-gate, нет follow-up
- "История ареста" -- dead end, нет возврата в воронку
- AI-виджет не имеет явного перехода к покупке
- Нет ретаргетинга по сегментам (скачал PDF, открыл форму, не завершил)

### 3.2 Рекомендуемые изменения

**Приоритет 1 (критично):**

1. **Вынести Telegram Bot API token из клиентского кода** -- сейчас token и chat_id видны любому в DevTools (Modals.tsx строка 16-17). Создать серверный endpoint (Cloudflare Worker, Vercel Function, или простой API) для отправки лидов. Это не только безопасность, но и возможность добавить:
   - Сохранение лидов в CRM/таблицу
   - Отправку email-подтверждения клиенту
   - Трекинг UTM-меток

2. **Добавить форму заявки для всех тарифов** -- сейчас форма только для "Брони". Добавить кнопку "Оставить заявку" рядом с каждой Telegram-ссылкой как альтернативу. Ожидаемый прирост конверсии: +20-40%.

3. **Email-gate для PDF** -- вместо прямой ссылки на PDF показывать мини-форму (email + имя). После ввода -- отправить PDF на email + добавить в email-цепочку. Это превратит мертвую метрику `pdf_download` в лид.

**Приоритет 2 (важно):**

4. **Exit-intent popup** -- при уходе со страницы предложить бесплатный мини-аудит. Формат: "Уходите? Получите бесплатный экспресс-аудит вашего сайта за 60 секунд". CTA ведет на AI-виджет.

5. **Sticky CTA на мобильных** -- плавающая кнопка "Заказать аудит" внизу экрана, видна всегда при скролле. На мобильных это дает +15-25% к конверсии.

6. **Блок отзывов/кейсов** -- добавить секцию между ROI и Pricing. Минимум 3 отзыва с фото, именем, компанией. Если реальных клиентов пока мало -- попросить первых 3-5 клиентов за скидку 20%.

7. **Счетчик аудитов** -- "Проверено 347+ сайтов" рядом с Hero или в Navbar. Даже начальное число создает доверие.

**Приоритет 3 (оптимизация):**

8. **Автооткрытие AI-виджета** через 45 секунд на странице с приветственным сообщением: "Хотите узнать, в безопасности ли ваш сайт? Спросите меня!"

9. **После "Истории ареста"** добавить CTA-оверлей внутри iframe-контейнера: "Не хотите повторить эту историю? Закажите аудит".

10. **A/B тест главного CTA** -- "Заказать аудит" vs "Проверить мой сайт бесплатно" vs "Узнать свой риск". Гипотеза: бесплатный вход конвертирует лучше.

### 3.3 Целевая воронка

## 6. Telegram-канал и бот

### 6.1 Telegram-канал

**Цель:** прогрев аудитории, экспертный контент, трафик на сайт и в бота.

**Контент-план (еженедельный цикл):**

- **Понедельник:** Новости/штрафы -- скриншот новости + краткий разбор + "А ваш сайт защищен?"
- **Среда:** Полезный контент -- чек-лист, инструкция, разбор конкретного пункта из 32
- **Пятница:** Кейс/разбор -- аудит публичного сайта, до/после клиента, или видео-кружок от Дениса

**Форматы:**
- Текст + изображение (карточка с цитатой закона или скриншот нарушения)
- Видео-кружки 30-60 сек (личный бренд Дениса)
- Опросы ("Есть ли на вашем сайте политика конфиденциальности?")
- Карусели (5-7 слайдов с чек-листом)

**Рост подписчиков:**
1. Ссылка с сайта (добавить в Navbar: "Наш канал")
2. CTA в каждой статье на VC.ru/Дзен: "Подпишитесь на канал, чтобы не пропустить штрафы"

5. Гостевые посты: Денис пишет экспертный пост для канала партнера

### 6.2 Telegram-бот (@WebAuditRuBot)

**Текущий функционал:** точка входа заявок через deep links (start=razvedka, start=proekt, start=zakazat_audit, start=sos_audit).

**Рекомендации по улучшению:**

1. **Автоматическая квалификация лида:**
   - После /start бот задает 3-4 вопроса:
     - "Какой у вас сайт?" (URL)
     - "Сколько форм сбора данных на сайте?"
     - "Есть ли политика конфиденциальности?"
     - "Какой примерный оборот бизнеса?"
   - На основе ответов рекомендует тариф

2. **Мгновенный мини-отчет:**
   - Бот автоматически проверяет URL (SSL, robots.txt, наличие политики конфиденциальности, мета-теги)
   - Выдает оценку A-F
   - "Ваш сайт получил оценку D. Хотите узнать все 32 уязвимости? Закажите полный аудит"

3. **Реферальная механика:**
   - Каждый пользователь получает уникальную реферальную ссылку
   - "Пригласите друга -- получите скидку 5 000 руб. на аудит"
   - Трекинг рефералов внутри бота

4. **Серия прогревающих сообщений:**
   - День 1: Мини-отчет
   - День 3: "Кстати, вот 3 нарушения, которые мы нашли на 90% сайтов"
   - День 7: "Новый штраф в [сфера клиента]. Ваш сайт в зоне риска?"
   - День 14: Оффер со скидкой 10% при заказе в течение 48 часов

5. **Интеграция с каналом:**
   - Бот автоматически постит в канал анонимизированную статистику: "За эту неделю проверено 47 сайтов. Средняя оценка: D+"

## 7. Вирусные механики

### 7.1 Бесплатный скоринг сайта (главный growth hack)

**Механика:**
1. Пользователь вводит URL (на сайте или в Telegram-боте)
2. Автоматическая проверка 5-7 параметров
3. Красивый отчет-карточка с оценкой A-F
4. Кнопка "Поделиться результатом" -- шеринг в TG/VK/Twitter
5. Текст шеринга: "Мой сайт получил оценку D по безопасности. Проверь свой: [ссылка]"

**Вирусный потенциал:** K-фактор 0.3-0.5 в B2B-среде (владельцы делятся с партнерами).

### 7.2 Бейдж безопасности

После прохождения аудита клиент получает виджет для своего сайта: "Проверено СайтЧИСТ. DS-CERT 2026". При клике -- переход на webaudit.ru. Каждый посетитель сайта клиента видит бейдж -- бесплатная реклама на постоянной основе.

### 7.3 Партнерский виджет

White-label виджет "Проверить безопасность сайта" для встраивания на сайты веб-студий. Каждая проверка генерирует лид для СайтЧИСТ, студия получает комиссию.
